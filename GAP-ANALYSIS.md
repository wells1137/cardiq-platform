## 三、差距清单（按优先级排序）

### 🔴 P0 — 核心数据层缺失（MVP 最关键）

#### 3.1 eBay 真实成交数据接入
**文档要求**：聚合 eBay 历史成交价（包括 Best Offer 隐藏真实成交价），这是平台核心价值。  
**现状**：后端使用**模拟随机数据**生成价格历史，无任何真实数据。  
**待开发**：
- 接入 eBay Browse API（需 App ID + Cert ID）
- 或接入 Apify eBay Sold Listings API（第三方合规爬虫服务，更易获取历史数据）
- 解析并还原 Best Offer 真实成交价
- 价格数据清洗与去重逻辑

#### 3.2 卡淘/闲鱼国内平台数据接入
**文档要求**：聚合国内卡淘成交价，提供人民币计价的"到手价"参考，这是对标 130point 的核心差异化。  
**现状**：完全没有国内数据源，所有价格均为美元模拟数据。  
**待开发**：
- 卡淘定向爬虫（需遵守 Robots 协议，仅采集公开成交信息）
- 闲鱼数据采集方案（可考虑淘宝开放平台接口）
- 汇率换算模块（USD → CNY，含运费/关税估算）
- 国内外价格双轨展示 UI

#### 3.3 数据清洗与标准化 SKU 引擎
**文档要求**：建立同义词库，将"LBJ"、"老詹"等非标名称映射到唯一标准 SKU。  
**现状**：无任何数据清洗逻辑，球员/卡片搜索依赖精确字符串匹配。  
**待开发**：
- 球员别名同义词库（数据库表）
- 卡片 SKU 标准化引擎（品牌/系列/年份/平行版本规范化）
- 模糊搜索支持（Elasticsearch 或 MySQL 全文索引）

---

### 🟠 P1 — 核心产品功能缺失（第二阶段必备）

#### 3.4 K线图（蜡烛图）
**文档要求**：将历史成交数据可视化为"类股票K线图"，直观展示涨跌趋势。  
**现状**：CardDetail 页面使用 Recharts 折线图，**不是 K 线图**。  
**待开发**：
- 引入 `lightweight-charts`（TradingView 开源库）或 `recharts` 自定义蜡烛图组件
- 后端新增 OHLC（开高低收）数据聚合接口（按日/周/月聚合成交价）
- 时间粒度切换（1D / 1W / 1M / 3M / 1Y / ALL）

#### 3.5 数字卡册（Portfolio 资产管理）
**文档要求**：用户录入自己持有的卡牌，系统实时计算总资产价值涨跌，这是 Card Ladder 的核心付费功能。  
**现状**：完全没有 Portfolio 功能，Watchlist 仅是监控列表，不记录持仓成本。  
**待开发**：
- 数据库新增 `portfolio` 表（持仓卡片、买入价、买入日期、数量）
- 资产总价值实时计算（持仓价 vs 当前市价）
- Portfolio 页面：总资产价值卡片、持仓列表、盈亏百分比、资产走势图
- 批量导入持仓（CSV 上传）

#### 3.6 PSA/BGS 人口报告（Pop Report）展示
**文档要求**：直观展示某张卡在特定分数下的存世量，帮助判断稀缺性。  
**现状**：Schema 中有 `population` 字段，但 UI 完全没有展示，也没有数据来源。  
**待开发**：
- 接入 PSA Population Report API（或爬取 psacard.com）
- CardDetail 页面新增"稀缺性分析"模块（各分数段存世量柱状图）
- 在卡片列表中显示 Pop 数量徽章

### 🟡 P2 — 商业化与增长功能（第三阶段）

#### 3.7 会员订阅系统（Pro 会员）
**文档要求**：基础查价免费（30天数据），Pro 会员 19.9元/月解锁无限历史数据、Portfolio、价格预警。  
**现状**：完全没有付费墙，所有功能免费开放。  
**待开发**：
- 接入 Stripe 支付（已有 `webdev_add_feature stripe` 支持）
- 用户角色扩展：`free` / `pro` / `admin`
- 数据访问权限控制（免费用户只看30天，Pro 看全量）
- 订阅管理页面（升级/降级/账单历史）

#### 3.8 AI 拍照识卡
**文档要求**：对标 CollX，用户拍照后 AI 自动识别卡牌信息并查价，降低新手门槛。  
**现状**：无此功能。  
**待开发**：
- 前端相机调用（`getUserMedia` API）
- 图片上传到 S3
- 后端调用 Gemini Vision API 识别卡牌品牌/系列/球员/年份
- 识别结果自动填入搜索框并跳转查价

#### 3.9 市场指数（Market Index）
**文档要求**：编制"詹姆斯指数"、"梅西指数"或"2020年新秀指数"，反映大盘走势。  
**现状**：无此功能。  
**待开发**：
- 指数定义系统（选定一组卡片作为指数成分）
- 指数历史走势计算与存储
- 市场大盘页面（多指数走势对比图）

#### 3.10 评级代送服务入口
**文档要求**：在查价页面嵌入 PSA/BGS 代送服务入口，赚取渠道差价。  
**现状**：无此功能。  
**待开发**：
- CardDetail 页面新增"申请评级"CTA 按钮
- 对接评级代送服务商（可先做简单的表单收集）

---

### 🔵 P3 — 运营与信任建设

#### 3.11 虚假交易识别算法
**文档要求**：识别"左手倒右手"刷单行为（价格异常偏离均值、短时间频繁交易），打上"可疑"标签。  
**现状**：无任何数据质量控制逻辑，所有价格数据直接入库。  
**待开发**：
- 异常价格检测算法（Z-score 偏离度检测）
- 可疑交易标记字段（`isSuspicious`）
- 数据清洗后的"已验证"价格标识

#### 3.12 UGC 成交记录上传
**文档要求**：建立用户自主上传成交记录的机制，降低对第三方平台数据的依赖。  
**现状**：无此功能。  
**待开发**：
- 用户上传成交截图功能
- AI 从截图中提取成交价格信息
- 社区贡献数据审核流程

## 四、技术架构改进建议

| 方向 | 现状 | 建议 |
|------|------|------|
| 搜索引擎 | MySQL LIKE 模糊查询 | 引入 Elasticsearch 或 MySQL 全文索引，支持同义词搜索 |
| 价格数据存储 | 单表 price_history | 增加 OHLC 聚合表，提升 K 线图查询性能 |
| 数据采集 | 无真实数据源 | 优先接入 Apify eBay API（最快路径），再自建爬虫 |
| 缓存层 | 无缓存 | 引入 Redis 缓存热门卡片价格，减少数据库压力 |
| 图片存储 | 无球员/卡片图片 | 接入 NBA 官方 CDN 球员头像，卡片图片用 S3 存储 |
| 支付 | 无 | Stripe（海外）+ 微信支付/支付宝（国内）双轨 |

## 六、现有代码可直接复用的部分

以下模块质量较高，可在 IDE 中直接在此基础上扩展，无需重写：

- **`server/sportsDataService.ts`** — BallDontLie API 集成 + 球员评分算法，可扩展接入更多运动 API
- **`server/cronScheduler.ts`** — Cron 调度引擎，可复用于定时价格抓取任务
- **`drizzle/schema.ts`** — 数据库 Schema 设计合理，只需新增 `portfolio`、`card_sku`、`price_ohlc` 等表
- **`client/src/pages/CardDetail.tsx`** — 卡片详情页框架，替换折线图为 K 线图即可
- **`client/src/pages/Dashboard.tsx`** — 2K 风格英雄区设计，可复用于 Portfolio 总览页
- **`server/routers.ts`** — tRPC 路由结构清晰，按模块拆分后可扩展性强
